## Countdown emoji break ‚Äî fixed
The `emoji` column from #198 **never actually landed in prod** (the owner's manual
`alter` didn't apply to project `aennreackkegaqwwbowg`). Effect: `countdown/page.tsx`
selected a missing column ‚Üí the query errored ‚Üí `data` null ‚Üí the list rendered
empty (looked like the redesign broke; the 12 rows were intact).
- **Fix:** applied `ALTER TABLE countdowns ADD COLUMN IF NOT EXISTS emoji TEXT` to prod
  via the **Supabase MCP** (advisors clean ‚Äî no new findings).
- **#201** hardened `page.tsx` + actions to degrade gracefully if the column is ever
  missing (select falls back to base columns; insert/update retry without `emoji`),
  so a missing migration can never blank the list again.
- **Lesson:** after a migration that new code depends on, VERIFY it landed
  (MCP `execute_sql` against `information_schema`) before merging the dependent code.
  The MCP `complete_authentication` "no flow" error is misleading ‚Äî it connects anyway.

## Design language reference (apply to every uplift)
Hero progress ring/blob (value-vs-goal or time elapsed-vs-remaining) ¬∑ fuzzy friendly
labels ¬∑ stat-header strip ¬∑ time/status bucketed sections with counts ¬∑ color-coded
status pills ¬∑ hue-tinted cards ¬∑ celebratory empty/complete states ¬∑ fully clickable
(inline add/edit/delete, bottom-sheet editors, filter chips) ¬∑ subtle motion. Keep dark/
zinc surfaces + a playful accent; STRONGLY prefer migration-free.
