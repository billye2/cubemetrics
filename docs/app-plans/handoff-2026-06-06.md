## Security audit + hardening (PR #145, applied to prod)
Ran a guardrail-focused audit (`get_advisors`). Fixed: `_seed_baseline` RLS-disabled+exposed (ERROR)
→ `ENABLE RLS`; `_audit_columns()` + `bump_app_usage()` were `PUBLIC`-executable SECURITY DEFINER fns
→ `REVOKE EXECUTE FROM PUBLIC` (revoking from anon/authenticated alone is insufficient — Postgres
grants to PUBLIC by default), `bump_app_usage` re-granted to `authenticated`. **Keep these guardrails:**
the spine `user_id`-filter invariant (digest cron runs adapters under the **service-role** client, so
`.eq("user_id")` is the only tenant isolation), fail-closed `isAdmin`, RLS-no-policy on
`app_admins`/`notification_log`/`_seed_baseline`, constant-time `CRON_SECRET` check, HMAC unsubscribe.
Also added a `deny` block to `.claude/settings.local.json` (`.env*`/settings/CI off-limits).
