## Repo hygiene — the `_generated.ts` line-ending fix (PR #177)
The catalog + spine `_generated.ts` had shown as modified in the working tree across sessions. It was
**never content drift** — the build scripts write **LF**, but `core.autocrlf=true` with no
`.gitattributes` made git expect **CRLF**, so every `npm run build` produced a zero-line phantom diff.
Added `.gitattributes` pinning `**/_generated.ts` to `text eol=lf`. Verified: a fresh
`build:catalog && build:spine` no longer dirties them. (Side effect: real `_generated.ts` content
changes — like the Stopwatch `hidden` flag — now stage as honest diffs.)
